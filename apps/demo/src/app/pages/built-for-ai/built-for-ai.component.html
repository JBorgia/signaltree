v10.2 — measured

AI-codegen accuracy benchmark

A reproducible matrix scoring how reliably each major AI agent generates correct Angular state code across libraries. Fixed prompt suite, idiomatic-pattern matchers, import-resolution and marker-method validators.

v10.2 expanded result (2026-05-29):

  • 49% — cold-run accuracy (no priming)
  • 91% — primed with /llms.txt
  • +42 percentage points from a single retrievable file
  • 99% — Claude Sonnet 4.6 primed (near-ceiling)

Across 6 models (Claude Sonnet 4.6, GPT-5.4, Gemini 3.1 Pro, Perplexity Sonar Pro, Claude Haiku 4.5, GPT-5.4-mini) on 8 prompts × 5 libraries × 3 priming modes = 720 cells. Scored on idiomatic pattern + import-resolution + marker-method validity.

Key finding: primed Haiku 4.5 (97/100) outscores cold Sonnet 4.6 (41/100) by 2.4×. Priming closes the model-tier gap — your code agent on a cheap model still writes correct SignalTree.

Strategic differentiator: every state library competes on bundle size and feature lists. SignalTree competes on AI-correctness percentages — and publishes the score.
