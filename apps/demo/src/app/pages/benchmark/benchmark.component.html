AI-codegen accuracy benchmark

How reliably do frontier AI coding agents generate correct Angular state code with each major state library? We measured 720 cells. Here is the scorecard.

  • 49% — SignalTree cold-run accuracy (no priming)
  • 91% — SignalTree primed with llms.txt
  • +42 percentage points from a single retrievable file
  • 99/100 — Claude Sonnet 4.6 primed (near ceiling)

Scope

8 prompts
5 libraries
6 agents
3 priming modes
~$12 in API spend

Models: Claude Sonnet 4.6, GPT-5.4, Gemini 3.1 Pro, Perplexity Sonar Pro (frontier tier); Claude Haiku 4.5, GPT-5.4-mini (cost tier). Prompts: counter, paginated users, debounced search, derived totals, login form, undo/redo, deeply nested status, multi-marker editor.

Key insights

Priming closes the model-tier gap

Cold scores are roughly tier-agnostic (range 41–56). Primed scores are also tier-agnostic (range 80–99).

Primed Haiku 4.5 (97) outscores cold Sonnet 4.6 (41) by 2.4×. Your code agent on a cheap model still writes correct SignalTree.

Claude Sonnet 4.6 hits the ceiling

Primed Claude scored 99/100 across all 8 prompts. Further improvement is at the prompt/scorer level, not the model level.

Adding myths.md regressed accuracy

We tested two priming compositions: llms.txt alone (91%) vs llms.txt + myths.md (87%). More context wasn't better — likely context dilution.

Gemini regressed sharply (83 → 64). Implication: prioritize focused priming over breadth.

The discoverability surface ships with npm

As of v10.2, llms.txt ships in the @signaltree/core tarball. After npm install, retrieval-aware AI agents find node_modules/@signaltree/core/llms.txt automatically.

The README also leads with the disambiguation table — rendered on npmjs.com and indexed by every AI tool that crawls node_modules READMEs.

Methodology

  • Cold mode: standard prompt + "you are a senior Angular engineer" system message. No SignalTree context.
  • Primed-llms.txt: the 13.5KB llms.txt injected as an additional system message.
  • Primed-llms.txt + myths.md: both files concatenated as priming context.
  • Reproducibility: all prompts as YAML in scripts/ai-codegen-benchmark/prompts/, temperature 0, model versions logged per cell.
  • Scoring: idiomatic-pattern match + import-resolution against real-package allowlist + marker-method validity against actual API surface. Equal-weighted average.

The marker-method validator catches the dominant primed-run failure mode (hallucinated method names) that text pattern-matching alone misses. Compile + behavior testing in TestBed is deferred to v10.3.

Reproduce it

The full benchmark is open and reproducible. One OpenRouter API key, ~$15 and ~30 minutes of wall-clock for a full quarterly run. 

git clone https://github.com/JBorgia/signaltree
cd signaltree
export OPENROUTER_API_KEY=sk-or-...

# Cold
node scripts/ai-codegen-benchmark/runner.mjs --include-tier-comparison

# Primed
PRIMING_CONTEXT_FILE=apps/demo/public/llms.txt \
  node scripts/ai-codegen-benchmark/runner.mjs --include-tier-comparison

# Aggregate
node scripts/ai-codegen-benchmark/aggregate.mjs \
  scripts/ai-codegen-benchmark/results/* > RESULTS.md
