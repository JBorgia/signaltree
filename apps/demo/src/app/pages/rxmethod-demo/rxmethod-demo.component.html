rxMethod

Encapsulated async pipelines for SignalTree — direct equivalent of NgRx's rxMethod. Same call shape, same input flexibility (raw value, Signal, or Observable), same auto-cleanup on DestroyRef. 

import { rxMethod } from '@signaltree/core/rxjs-interop';

What you're seeing

  • Auto-cleanup. Every subscription (master pipeline, signal bridge, observable bridge) auto-unsubscribes when this component's DestroyRef fires. No manual ngOnDestroy wiring.
  • Input flexibility. rxMethod<string> accepts a raw string, a Signal<string>, or an Observable<string>. Demo 2 wires a signal in ngOnInit and just calls .set() on the leaf to push values through.
  • No store ceremony. The pipeline writes directly to tree paths via this.store.$.path.set(). Race conditions, cancellation, debounce — all standard RxJS, no library-specific abstractions.

API shape

readonly loadUsers = rxMethod<void>((input$) =>
  input$.pipe(
    tap(() => this.store.$.loading.setLoading()),
    switchMap(() => this.api.list$().pipe(
      tap((users) => this.store.$.users.entities.setAll(users)),
      tap(() => this.store.$.loading.setLoaded()),
      catchError((err) => {
        this.store.$.loading.setError(err);
        return EMPTY;
      })
    ))
  )
);
