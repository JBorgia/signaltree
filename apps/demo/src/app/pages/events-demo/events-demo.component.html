State Reactions — tree.effect()

Don't confuse @signaltree/events with tree.effect(). The events package is an external typed event bus for cross-system communication. tree.effect() is the REACT pillar of the 3-pillar architecture — it runs whenever state changes, with no actions or dispatch required.

@signaltree/events — external event bus

For typed, validated, idempotent events crossing system or service boundaries.

tree.effect() — state-reactive side effects

For reacting to app state changes within the same Angular app. No events, no dispatch.

When to use which:
  • Typed, cross-service/cross-app message → @signaltree/events
  • React to in-app state change (filter, auth, selection) → tree.effect()
  • Side effect triggered by user action → write state in Ops, react with tree.effect()
}
