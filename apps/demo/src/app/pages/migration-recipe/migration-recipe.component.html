Migration Recipe & Side-by-Side Comparisons

Six progressively more complex feature implementations, each shown side-by-side: SignalTree on the left, the comparison library you pick on the right. Each example targets the same capability checklist, and within an example both panes are written at the same architectural level — a compact single-file service compares against a compact single-file store; a multi-file AppTree/Ops/AppStore layout compares against the equivalent multi-store + facade layout.

What we don't claim: that the libraries' internals are equivalent. SignalTree's status(), entityMap(), stored(), batching(), devTools(), and timeTravel() primitives encapsulate behavior that other libraries reconstruct from primitives in app code. That's the point of the comparison — not "look how much shorter SignalTree is" but "look at what each library asks your code to express."

One more honest note: SignalTree can be as small as you want it to be. The APP_TREE + provideAppTree() + AppStore + per-domain Ops layout shown in Example 6 is a recommended architecture for multi-domain apps — it is not mandatory. Examples 1–5 use plain @Injectable services owning a signalTree directly, which is the same scale as a signalStore or a single BehaviorSubject service. Pick the architecture that fits your app's complexity.

1. Counter — state + one derived + one mutation

The smallest possible feature. One number, one derived value (doubled), one increment method. This is where the mechanical overhead of each library is most visible.

Capability checklist (identical for every example):
  • count: number state
  • doubled: number derived value
  • increment() mutation
  • Component reads count + doubled as signals and calls increment()

SignalTree

// counter.service.ts
import { Injectable, computed } from '@angular/core';
import { signalTree } from '@signaltree/core';

@Injectable({ providedIn: 'root' })
export class CounterService {
  private tree = signalTree({ count: 0 })
    .derived(($) => ({
      doubled: computed(() => $.count() * 2),
    }));

  count = this.tree.$.count;       // Signal<number>
  doubled = this.tree.$.doubled;   // Signal<number>

  increment() {
    this.tree.$.count.update((n) => n + 1);
  }
}

2. Cart — JSON state + multiple derived values

A cart with a list of line items, a derived count, a derived total, an add() mutation, and a remove() mutation.

Capability checklist:
  • items: CartItem[] state (each item has id, price, qty)
  • count derived = total quantity
  • total derived = price × quantity sum
  • add(item), remove(id) mutations
  • Component reads items, count, total

SignalTree

// cart.types.ts
export interface CartItem { id: string; price: number; qty: number; }

// cart.service.ts
import { Injectable, computed } from '@angular/core';
import { signalTree } from '@signaltree/core';
import { CartItem } from './cart.types';

@Injectable({ providedIn: 'root' })
export class CartService {
  private tree = signalTree({ items: [] as CartItem[] })
    .derived(($) => ({
      count: computed(() => $.items().reduce((n, i) => n + i.qty, 0)),
      total: computed(() => $.items().reduce((s, i) => s + i.price * i.qty, 0)),
    }));

  items = this.tree.$.items;
  count = this.tree.$.count;
  total = this.tree.$.total;

  add(item: CartItem) {
    this.tree.$.items.update((xs) => [...xs, item]);
  }

  remove(id: string) {
    this.tree.$.items.update((xs) => xs.filter((x) => x.id !== id));
  }
}

import { CartItem } from './cart.types';
import { AddToCart, RemoveFromCart } from './cart.actions';

@State<CartStateModel>({
  name: 'cart',
  defaults: { items: [] },
})
@Injectable()
export class CartState {
  @Selector() static items(s: CartStateModel) { return s.items; }
  @Selector([CartState.items])
  static count(items: CartItem[]) {
    return items.reduce((n, i) => n + i.qty, 0);
  }
  @Selector([CartState.items])
  static total(items: CartItem[]) {
    return items.reduce((s, i) => s + i.price * i.qty, 0);

  add(ctx: StateContext<CartStateModel>, { item }: AddToCart) {
    ctx.patchState({ items: [...ctx.getState().items, item] });

    ctx.patchState({
      items: ctx.getState().items.filter((x) => x.id !== id),

3. Async data — loading / error / data

Fetch a user profile from an API. Track loading state and errors, expose data, allow retry. This is the smallest realistic single-resource async feature.

Capability checklist:
  • profile: Profile | null
  • isLoading: boolean, error: string | null
  • load() async fetch
  • retry() re-runs the load

SignalTree (using the status() marker)

// profile.types.ts
export interface Profile { id: string; name: string; email: string; }

// profile.service.ts
import { Injectable, inject } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { firstValueFrom } from 'rxjs';
import { signalTree, status } from '@signaltree/core';
import { Profile } from './profile.types';

@Injectable({ providedIn: 'root' })
export class ProfileService {
  private http = inject(HttpClient);

  private tree = signalTree({
    profile: null as Profile | null,
    load: status(),
  });

  profile    = this.tree.$.profile;
  isLoading  = this.tree.$.load.isLoading;
  error      = this.tree.$.load.error;

  async load(): Promise<void> {
    this.tree.$.load.start();
    try {
      const data = await firstValueFrom(
        this.http.get<Profile>('/api/me')
      );
      this.tree.$.profile.set(data);
      this.tree.$.load.success();
    } catch (err) {
      this.tree.$.load.fail((err as Error).message);
    }
  }

  retry(): Promise<void> { return this.load(); }
}

4. CRUD entity collection with filter

The production pattern: a normalized collection of tickets, a text filter, loading/error tracking, async load, optimistic update with rollback on failure, and a derived visible list.

Capability checklist:
  • Normalized entity collection of Ticket
  • filter: string
  • visible = entities filtered by filter
  • Loading + error tracking
  • Async load()
  • Optimistic updateTitle(id, title) with rollback on failure

SignalTree (single-service form — matches the signalStore scale)

// tickets.types.ts
export interface Ticket { id: string; title: string; done: boolean; }

// tickets.service.ts
import { Injectable, computed, inject } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { firstValueFrom } from 'rxjs';
import { signalTree, entityMap, status } from '@signaltree/core';
import { Ticket } from './tickets.types';

@Injectable({ providedIn: 'root' })
export class TicketsService {
  private http = inject(HttpClient);

  private tree = signalTree({
    entities: entityMap<Ticket, string>(),
    filter: '',
    load: status(),
  }).derived(($) => ({
    visible: computed(() => {
      const f = $.filter().toLowerCase();
      return $.entities.all().filter((t) =>
        t.title.toLowerCase().includes(f)
      );
    }),
  }));

  visible   = this.tree.$.visible;
  isLoading = this.tree.$.load.isLoading;
  error     = this.tree.$.load.error;

  setFilter(filter: string) { this.tree.$.filter.set(filter); }

  async load(): Promise<void> {
    this.tree.$.load.start();
    try {
      const list = await firstValueFrom(
        this.http.get<Ticket[]>('/api/tickets')
      );
      this.tree.$.entities.setAll(list, { selectId: (t) => t.id });
      this.tree.$.load.success();
    } catch (err) {
      this.tree.$.load.fail((err as Error).message);
    }
  }

  async updateTitle(id: string, title: string): Promise<void> {
    const previous = this.tree.$.entities.byId(id)?.();
    if (!previous) return;

    this.tree.$.entities.upsertOne({ ...previous, title });

    try {
      await firstValueFrom(
        this.http.patch('/api/tickets/' + id, { title })
      );
    } catch (err) {
      this.tree.$.entities.upsertOne(previous);
      this.tree.$.load.fail((err as Error).message);
    }
  }

  retry(): Promise<void> { return this.load(); }
}

5. Reactive cross-domain effect — selection drives a fetch

When selectedUserId changes, fetch that user's orders. Cancel any in-flight request for the previous user. Clear orders when the selection becomes null.

Capability checklist:
  • Shared selectedUserId: string | null
  • orders: Order[], ordersLoading: boolean
  • Selection change cancels prior request (no race)
  • Selection of null clears orders

SignalTree (single-service form — effect() + AbortController)

// orders.types.ts
export interface Order { id: string; userId: string; total: number; }

// users.service.ts
import { Injectable, effect, inject, untracked } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Subscription } from 'rxjs';
import { signalTree } from '@signaltree/core';
import { Order } from './orders.types';

@Injectable({ providedIn: 'root' })
export class UsersService {
  private http = inject(HttpClient);

  private tree = signalTree({
    selectedUserId: null as string | null,
    orders:        [] as Order[],
    ordersLoading: false,
  });

  selectedUserId = this.tree.$.selectedUserId;
  orders         = this.tree.$.orders;
  ordersLoading  = this.tree.$.ordersLoading;

  constructor() {
    let sub: Subscription | null = null;

    effect(() => {
      const userId = this.tree.$.selectedUserId();

      sub?.unsubscribe(); // cancel any in-flight request

      if (!userId) {
        untracked(() => {
          this.tree.$.orders.set([]);
          this.tree.$.ordersLoading.set(false);
        });
        return;
      }

      untracked(() => this.tree.$.ordersLoading.set(true));

      sub = this.http.get<Order[]>('/api/users/' + userId + '/orders')
        .subscribe({
          next: (orders) => {
            this.tree.$.orders.set(orders);
            this.tree.$.ordersLoading.set(false);
          },
          error: (err) => {
            console.error('orders load failed', err);
            this.tree.$.ordersLoading.set(false);
          },
        });
    });
  }

  selectUser(id: string | null) {
    this.tree.$.selectedUserId.set(id);
  }
}

// tree/app-tree.ts
import { InjectionToken, Provider, computed } from '@angular/core';
import { signalTree, batching, devTools, timeTravel } from '@signaltree/core';
import { identityState } from '../identity/identity.state';
import { ticketsState } from '../tickets/tickets.state';
import { settingsState } from '../settings/settings.state';

export function createBaseState() {
  return {
    identity: identityState(),
    tickets:  ticketsState(),
    settings: settingsState(),
  };
}

export function createAppTree() {
  return signalTree(createBaseState())
    .with(devTools({ treeName: 'AppTree' }))
    .with(batching())
    .with(timeTravel())
    // Tier 1: entity resolution
    .derived(($) => ({
      tickets: {
        active: computed(() => {
          const id = $.tickets.activeId();
          return id ? $.tickets.entities.byId(id)?.() ?? null : null;
        }),
        all: computed(() => $.tickets.entities.all()),
      },
    }))
    // Tier 2: cross-domain UI rollups
    .derived(($) => ({
      ui: {
        isAuthenticated: computed(() => $.identity.user() !== null),
        openTicketsForUser: computed(() => {
          if (!$.identity.user()) return [];
          return $.tickets.all().filter((t) => !t.done);
        }),
      },
    }));
}

export type AppTree = ReturnType<typeof createAppTree>;
export const APP_TREE = new InjectionToken<AppTree>('APP_TREE');
export function provideAppTree(): Provider[] {
  return [{ provide: APP_TREE, useFactory: () => createAppTree() }];
}

    async updateTitle(id: string, title: string) {
      const previous = store.entityMap()[id];
      if (!previous) return;
      patchState(store, updateEntity({ id, changes: { title } }));
      try {
        await firstValueFrom(http.patch('/api/tickets/' + id, { title }));
      } catch (err) {
        patchState(
          store,
          updateEntity({ id, changes: { title: previous.title } }),
          { error: (err as Error).message },
        );
      }
    },

    /** Wipes all tickets (called from AppStore.logout()). */
    clearAll() {
      patchState(store, removeAllEntities(), { activeId: null });
    },
  }))
);

// settings.store.ts
import { effect } from '@angular/core';
import {
  patchState, signalStore, withHooks, withMethods, withState,
} from '@ngrx/signals';

interface Settings {
  theme: 'light' | 'dark';
  locale: string;
}
function load(): Settings {
  return {
    theme: (localStorage.getItem('app.theme') as 'light' | 'dark') ?? 'light',
    locale: localStorage.getItem('app.locale') ?? 'en-US',
  };
}

export const SettingsStore = signalStore(
  { providedIn: 'root' },
  withState(load()),
  withMethods((store) => ({
    setTheme(theme: 'light' | 'dark') { patchState(store, { theme }); },
    setLocale(locale: string)         { patchState(store, { locale }); },
  })),
  withHooks({
    onInit(store) {
      // manual persistence — equivalent to SignalTree's stored() marker
      effect(() => localStorage.setItem('app.theme', store.theme()));
      effect(() => localStorage.setItem('app.locale', store.locale()));
    },
  }),
);

import { Injectable, inject, computed } from '@angular/core';
import { IdentityStore } from './identity.store';
import { TicketStore } from './tickets.store';
import { SettingsStore } from './settings.store';

@Injectable({ providedIn: 'root' })
export class AppStore {
  readonly identity = inject(IdentityStore);
  readonly tickets  = inject(TicketStore);
  readonly settings = inject(SettingsStore);

  // Cross-store derived value — re-implemented on the facade because
  // signalStore doesn't expose a way to derive across stores natively.
  readonly openTicketsForUser = computed(() => {
    if (!this.identity.isAuthenticated()) return [];
    return this.tickets.entities().filter((t) => !t.done);
  });

  /** Cross-store orchestration. */
  logout() {
    this.identity.clear();
    this.tickets.clearAll();
    // No batching across stores — two separate change-detection ticks.
  }
}

Takeaway: SignalTree composes one tree from domain-state factories, derives across domains in one place, and gets cross-cutting concerns (DevTools, batching, time travel, persistence) on the whole tree with one .with(...) call per concern. @ngrx/signals delivers the same capabilities domain-by-domain, but you re-implement persistence in every store, derive-across-stores happens on the facade rather than the data graph, and there's no first-class batching across stores.

AI-assisted migration — the SignalTree Skill

SignalTree ships a vendor-neutral Agent Skill at node_modules/@signaltree/core/skills/using-signaltree/. It works with Cursor, Claude Code, GitHub Copilot, or any SKILL.md-aware AI assistant. The skill is what an agent reads to learn SignalTree's mental model, the canonical AppTree + AppStore + Ops shape, and the playbook for migrating an existing codebase off another state library.

One-time install per repo

# Cursor
cp -r node_modules/@signaltree/core/skills/using-signaltree .cursor/skills/

# Claude Code
cp -r node_modules/@signaltree/core/skills/using-signaltree .claude/skills/

# GitHub Copilot (custom instructions / agent customizations)
cp -r node_modules/@signaltree/core/skills/using-signaltree \
  .github/skills/

What the skill teaches the agent

  • Mental model. Reactive JSON tree, the $ proxy, leaves as WritableSignal<T>, markers (entityMap, status, stored, form), enhancer composition order.
  • Canonical architecture. One tree per app, APP_TREE + provideAppTree() + provideAppTreeForTesting() from day one, AppStore facade, per-domain Ops, the strict consumer rule (components inject AppStore only).
  • Tier ladder for derived signals. Tier 0 base → 1 entity resolution → 2 complex logic → 3 workflow → 4 navigation → 5 UI aggregates, with named tier files (tier-entity-resolution.derived.ts, etc.) once the tree crosses ~3 domains.
  • Migration playbook. Big-bang is the default (delete the legacy package in the same PR). The hybrid legacy-facade pattern is a temporary fallback that must ship with a // TODO(legacy-facade): remove by <date> and a tracking issue.
  • Deepest coverage today is for @ngrx/signals — see reference/migration-from-ngrx-signals.md inside the skill. PRs welcome to extend coverage to classic NgRx, NGXS, Elf, and the others shown above.
  • Definition of done. (1) Zero imports of the legacy package in the migrated app. (2) Legacy package removed from package.json (or tracked for removal). (3) Test suite green — not just build-green. (4) DevTools shows the new tree under the chosen treeName.
  • Anti-pattern catalog. Multiple competing trees, root-injected Ops with eager side-effects in constructors, mocking AppStore in tests instead of providing APP_TREE, etc.
  • Orchestration mode. A separate playbook (reference/orchestrating-a-migration.md) for an orchestrator agent driving phased rollout across multiple implementer subagents. Load when the work spans more than ~5 files or when one agent will exhaust its context window.

Workflow: point your AI assistant at the skill, give it a target store to migrate, and it will follow the playbook end-to-end — including running build, test, lint, and asserting the legacy import is gone from application code and package.json before declaring done. See Using SignalTree with AI Agents for harness-specific setup.
