Migration Recipe: NgRx to SignalTree

This page walks through a small, practical migration from NgRx Signals to SignalTree for a single domain.

Step 1: Identify the store

// Example NgRx signals store (simplified)
export const TicketStore = signalStore(
  { providedIn: 'root' },
  withState<TicketState>({ /* ... */ }),
  withComputed(...),
  withMethods(...),
  withEntities(),
  withReduxDevtools()
);

Step 2: Map concepts

  • State → add to `app-tree`
  • Computed → `AppComputed` service
  • Methods → `ops` service
  • Entities → `entityMap()`
  • Devtools → `devTools()` enhancer

Step 3: Replace in small PRs

  1. Extract computed values to `AppComputed` and add tests.
  2. Move methods to an Ops service and update callers.
  3. Replace entity adapter with `entityMap()`.
  4. Remove old store files and run `pnpm test` and `pnpm nx build --statsJson` to compare bundle.
