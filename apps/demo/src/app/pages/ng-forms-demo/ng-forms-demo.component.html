SignalTree + Angular Forms

Tree-structured signal forms for Angular 22+. This demo showcases why @signaltree/ng-forms is essential for complex, production-ready forms—going far beyond what Angular 22's native FormField<T> provides.

🌲 Why use ng-forms over Angular 22 signal forms?

  • Tree Integration: Form state lives inside your SignalTree store (entities, async state, and forms in one tree) with typed path access—Angular 22's FieldTree nests fine but is a standalone model, not part of your store
  • Auto-Persistence: Built-in localStorage/sessionStorage with debouncing—no manual saving needed
  • Wizard Support: Multi-step forms with automatic field visibility—would require custom implementation in Angular 22
  • History Tracking: Undo/redo out of the box—not available in Angular 22
  • Reactive Bridge: Works with existing FormGroup/FormControl—Angular 22 is a new API requiring migration
  • Declarative Config: Centralized fieldConfigs with glob patterns (payment.card.*)—Angular 22 requires per-field setup

Use both! Angular 22 FormField for simple login forms. ng-forms for nested checkout flows, wizards, and forms with persistence. Want them wired together on one field? See the /signal-forms demo for the native-interop bridge (signalForm()).

🔗 Reactive Forms Bridge

This demo uses [formGroup]="profile.form" and formControlName directives—the same reactive forms API you already know. ng-forms provides both signals (profile.$.name()) and reactive controls (profile.form), enabling incremental migration.

Angular 22's FormField is a new API requiring rewrite. ng-forms lets you keep existing templates while adding signal capabilities.

Angular 22 vs ng-forms

Feature Angular 22 ng-forms
Nested objects ❌ Flat only ✅ Tree structure
Persistence ❌ Manual ✅ Built-in
Wizard flows ❌ Custom ✅ First-class API
History/undo ❌ Not available ✅ Built-in
Reactive bridge ⚠️ Separate API ✅ FormGroup compatible
Declarative config ⚠️ Per-field ✅ Centralized + globs

Both are valuable! Use Angular 22 for simple forms. Use ng-forms for complex enterprise scenarios.

🧙 Need Multi-Step Forms?

This demo shows a single-page form. For wizard flows (onboarding, checkout), use createWizardForm(): 

const wizard = createWizardForm([
            {{ '{' }} fields: ['profile.name', 'profile.email'] {{ '}' }},
            {{ '{' }} fields: ['company.name', 'company.size'] {{ '}' }},
            {{ '{' }} fields: ['preferences.*'] {{ '}' }}
            ], initialValues);

            wizard.nextStep();  // Auto-shows next fields
            wizard.previousStep();
          wizard.currentStep(); // Signal

Angular 22 has no wizard support. You'd build step management manually.
