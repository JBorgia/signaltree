SignalTree v2: Consolidated Architecture

Experience the power of a unified, optimized state management solution that combines all features into a single, tree-shakable package.

üéØ Why Consolidation Matters

Stop choosing between features and bundle size. Traditional state management forces you to pick: install separate packages for every feature (NgRx: @ngrx/store, @ngrx/effects, @ngrx/entity, @ngrx/router-store, @ngrx/component-store...) or settle for limited functionality.

üì¶ 85% Smaller Bundles

Tree-shakable architecture means you only ship what you use. Import just signalTree() for basic state (0.8KB), add withComputed() for derived values (+0.3KB), or withHistory() for time-travel (+0.4KB). No forced dependencies.

üîß Zero Version Conflicts

NgRx's separate packages create peer dependency hell. Need @ngrx/store@16.x with @ngrx/effects@16.x and @ngrx/entity@16.x? Hope they're compatible. SignalTree: one package, one version, zero conflicts.

‚ö° Optimized Together

Separate packages can't share optimizations. SignalTree's batching, memoization, and lazy evaluation work across all features because they're designed as a unified system.

Real example: NgRx with effects, entity, devtools = ~45KB. SignalTree with computed, history, persistence = ~3.4KB. 92% reduction.
