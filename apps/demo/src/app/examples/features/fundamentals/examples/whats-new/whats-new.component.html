� v9.0.0 — API Sharpening Release (April 2026)

SignalTree v9 aligns the API surface with the core ethos: state is data, use Angular directly, ship less, mean more.

⚠️ Breaking Changes

  • 37 deprecated exports removed: Entity aliases (entities(), enableEntities()), async helpers (createAsyncOperation(), trackAsync()), devtools variants (enableDevTools(), fullDevTools(), productionDevTools()), and more. Use the unified factory functions instead.
  • 7 memoization variants consolidated: shallowMemoization(), lightweightMemoization(), etc. are now memoization({ preset: 'shallow' }).
  • Subpath exports: Security, presets, storage, and edit-session moved to @signaltree/core/security, /presets, /storage, /edit-session.

✨ New in v9

  • Lifecycle cleanup contract: tree.destroyed readonly signal and registerCleanup(fn) for deterministic resource disposal.
  • Enhancer composition safety: Duplicate enhancer detection via ENHANCER_META symbol — applying the same enhancer twice now throws in dev mode.
  • CI budget enforcement: Bundle size, export count, and tree-shaking verified on every commit.

See the v8 → v9 migration guide for before/after code and the full list of changes.

⚠️ v6.0.0 — Breaking Changes (summary)

SignalTree v6 tightens types and makes enhancers opt-in. Important user-facing changes (short summary):

  • Enhancer factories renamed: Short factories are now primary (e.g. persistence()); legacy withX() names remain as deprecated aliases.
  • Methods are added by enhancers: The base tree no longer declares all methods — apply an enhancer to gain its methods (e.g. time-travel, batching). This makes the type system reflect runtime behavior.
  • Polymorphic enhancer pattern: Enhancers are now polymorphic in the tree state type so TypeScript correctly infers updated APIs when you call .with().
  • Marker-based entity API: Use entityMap() in your initial state (marker) and access reactive CRUD via store.$.myEntities.*.
  • Typing & API cleanup: Removed index-signature workarounds and improved inference—prefer annotating initial values with per-field assertions (see Preferred Typing docs).

See packages/core/UPGRADE_TO_V6.md and ENHANCER_TYPE_SYSTEM_V6.md for the full migration guide and examples.

🔥 v5.0.0 (December 2025) - Major Release

⚠️ Breaking Changes

  • Middleware System Removed: The entire middleware system has been deleted. Use tap() and intercept() hooks on entity signals instead for lifecycle observation.
  • subscribe() throws outside injection context: Calling tree.subscribe() outside Angular's injection context now throws an Error instead of silently failing. Enhancers use polling fallback when needed.

✨ New Features

  • PathNotifier System: Internal notification system for path-based mutations. Enables entity hooks without global state.
  • Entity Hooks - tap(): Observe entity mutations (add/update/remove) after they occur. 
    store.$.users.tap({
  onAdd: (user, id) => console.log('Added:', user),
  onUpdate: (id, changes) => console.log('Updated:', id),
  onRemove: (id, user) => console.log('Removed:', user),
});
  • Entity Hooks - intercept(): Block or transform mutations before they happen. Perfect for validation! 
    store.$.users.intercept({
  onAdd: (user, ctx) => {
    if (!user.email.includes('@')) {
      ctx.block('Invalid email');
    }
  },
  onUpdate: (id, changes, ctx) => {
    ctx.transform({ ...changes, updatedAt: new Date() });
  },
});
  • Marker-Based Entity System: Type-safe EntityMapMarker<T, ID> with reactive CRUD operations via store.$.field.method()
  • Enhanced Types: Recursive inference up to 20+ nesting levels with improved IntelliSense
  • Angular 20.3 Support: Full compatibility with signal forms using connect() and toWritableSignal()
  • Tree-Shakeable Audit: Import audit tracking separately via @signaltree/ng-forms/audit

🚀 Performance Improvements

  • 2-5x faster bulk operations with diff-based updates in enterprise enhancer
  • 40.5% bundle size reduction with consolidated architecture and shared dependencies
  • Zero polling in Angular: subscribe() uses Angular effects when available, falling back to polling only in tests

v4.2.0 (November 2025)

  • 🎯 Nested Entity Paths: Access deeply nested entities with dot notation! tree.entities<User>('app.data.users') enables domain-driven state organization.
  • Type-Safe Deep Paths: Full TypeScript inference for nested paths up to 5 levels deep with compile-time validation.
  • Removed asyncAction: Non-functional stub removed. Use manual async patterns with try/catch for better control and error handling.
  • Performance: Path resolution memoized for zero overhead after initialization.
