⚡ Async Operations

Async state in SignalTree is a marker family, not a free-standing service method. Two markers cover the two main shapes:

  • asyncSource<T>(config) — load-and-expose. Auto-loads on materialization. Exposes data, loading, error, plus refresh / set / update / reset. Accepts Observable or Promise loaders.
  • asyncQuery<TInput, TResult>(config) — input-driven debounced query. Writable input signal drives a built-in debounce → filter → distinctUntilChanged → switchMap pipeline. Race conditions, cancellation, dedup all handled automatically.

Both markers attach at any tree depth alongside entityMap, status, stored, and form. Auto-clean on the surrounding DestroyRef. No manual tap() / setLoading() wiring.

👉 Full interactive demo: /async — exercise both markers end-to-end with simulated network latency, debounced search, and live counters showing how cancellation and dedup work in practice. 

import { signalTree, asyncSource, asyncQuery } from '@signaltree/core';

const store = signalTree({
  users: asyncSource<User[]>({
    initial: [],
    load: () => this.api.list$(),
  }),
  search: asyncQuery<string, User[]>({
    initialResult: [],
    debounce: 300,
    filter: (q) => q.length > 0,
    query: (q) => this.api.search$(q),
  }),
});

// Uniform reads:
store.$.users();           // User[] | undefined
store.$.users.loading();   // boolean
store.$.users.error();     // unknown | null
store.$.users.refresh();   // reload (cancels in-flight)

store.$.search.input.set('alice');  // drives debounced pipeline
store.$.search();                    // results
