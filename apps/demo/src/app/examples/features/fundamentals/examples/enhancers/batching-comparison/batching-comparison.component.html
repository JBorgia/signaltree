Compare N direct signal writes against the same writes grouped with coalesce() from the batching() enhancer. Coalescing dedupes same-path writes, so only the final value is applied to the underlying signal. Note that Angular's effect scheduler already coalesces renders for synchronous writes — the render count is similar in both modes; what batching saves is the per-write signal-graph work and subscriber notifications.

"Writes Applied" counts writes that reached the underlying signal. Unbatched applies all N; coalesce applies only the last value per path. "Effect Runs" counts effect() executions — Angular batches these on the microtask in both modes, which is exactly why per-write renders are not the metric to compare.
