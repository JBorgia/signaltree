🔬 Effect Reactivity Test

This test replicates the exact pattern from the v3 trax-mobile app. It verifies that effect() correctly re-runs when SignalTree signals change, exactly as it does for regular Angular signals.

✅ What This Verifies

SignalTree signals do trigger Angular's effect() re-execution. Click the buttons below and watch every effect counter increment — nested primitives and entity signals behave identically to regular Angular signals.

Local-dev caveat: Under pnpm link local development you may hit a "dual Angular instance" issue where SignalTree's signals are created with a different @angular/core than the consuming app, causing isSignal() to return false and breaking effect reactivity. This never affects npm installs. See docs/guides/local-development-symlinks.md.
